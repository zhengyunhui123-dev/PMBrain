# PMBrain 桌面版安装与首次使用

本文面向第一次使用 PMBrain 的 Windows 用户，也适用于已经使用命令行版、Docker Postgres 或旧版 `.gbrain` 配置的升级用户。

## 先说结论

如果选择 **PGLite 零配置模式**，只需要安装 `PMBrain-Windows-x64-Setup-1.0.23.exe`，不需要另外安装 Bun、Node.js、Docker 或 Postgres。安装包已经包含 PMBrain sidecar、PGLite 数据库和所需的 WASM 运行时。

如果选择 **Docker Postgres 模式**，PMBrain 安装包只负责连接数据库，不负责安装或启动 Docker。使用前需要自行准备可访问的 Postgres/pgvector 数据库。

## 安装前条件

### 所有用户

- Windows 10 或 Windows 11，64 位系统。
- 建议至少预留 1 GB 可用磁盘空间，知识库数据另计。
- 当前安装包尚未配置正式代码签名，Windows SmartScreen 可能显示未知发布者提示。
- 如果需要调用在线模型或自动更新，需要允许 PMBrain 访问对应模型服务和 GitHub。

### PGLite 模式

没有额外前置软件。数据库文件保存在本机，适合个人使用、首次体验和中小型知识库。

### Docker Postgres 模式

需要提前准备：

- Docker Desktop 或其他可用的 Postgres 服务。
- 已启用 `pgvector` 的数据库。
- 完整连接地址，例如 `postgresql://用户名:密码@127.0.0.1:5433/pmbrain`。
- 数据库服务已启动，并允许本机连接。

## 新用户安装

1. 双击 `PMBrain-Windows-x64-Setup-1.0.23.exe`。
2. 选择安装目录并完成安装。
3. 启动 PMBrain。只有本机不存在 PMBrain 配置时，才会出现首次配置向导。
4. 选择数据库模式：
   - 不想安装其他软件：选择 **PGLite 零配置**。
   - 已有 pgvector Postgres：选择 **Docker Postgres** 并填写连接地址。
5. 选择知识库目录。PMBrain 会将它注册为独立 Source，后续可以同步、归档和恢复。
6. 按需要填写模型 API Key，然后点击“保存配置并启动”。
7. 数据库初始化完成后，桌面端会进入 PMBrain 管理台。

初始化期间不要重复启动程序。首次创建 PGLite 数据库或执行迁移可能需要一些时间。

## 模型 API Key 怎么填

API Key 不是启动 PMBrain 的硬性条件。不填写时仍可启动、管理 Source 和使用不依赖模型的能力，但向量化、对话、搜索扩展等 AI 功能会受到限制。

桌面向导支持 MIMO、智谱 AI、DeepSeek、OpenAI、Anthropic 和 ZeroEntropy。可以先只填写一个当前可用的提供商，后续再从桌面设置或管理台补充。

已有 Key 的用户不需要重复填写。输入框显示“已配置，留空保留”时，留空不会删除原值。

## 老用户升级

桌面端按以下顺序读取本机配置：

1. `%USERPROFILE%\.pmbrain\config.json`
2. `%USERPROFILE%\.gbrain\config.json`（旧版兼容）

检测到任一现有配置后，PMBrain 会：

- 沿用数据库模式、数据库地址和模型 API Key。
- 沿用知识库目录和 MCP/API 凭证。
- 仅在桌面版本变化时执行一次幂等数据库迁移。
- 迁移成功后启动 sidecar、完成健康检查并直接进入管理台。

老用户正常启动时不需要经过配置向导，也不需要点击“保存配置并启动”。只有从 PMBrain 菜单主动选择“配置与 MCP 接入”时，才会进入设置页。

如果启动失败，桌面端会显示独立的恢复页。建议先点击“重新启动服务”或打开日志，不要重新填写已有配置。

## MCP 接入是否必须

不是。MCP 接入只用于让 CodeBuddy、Cursor、Claude Code 或 Codex 调用 PMBrain。

需要时进入“配置与 MCP 接入”页面：

- CodeBuddy、Cursor、Codex：桌面端可以备份并合并配置。
- Claude Code：桌面端生成可复制的接入命令。
- API Key 模式会执行 MCP smoke test。
- OAuth 模式会生成独立 client credentials。

## 自动更新

正式 GitHub Release 发布后，桌面端会在启动后自动检查更新：

1. 发现新版本后自动下载。
2. 下载完成后提示安装。
3. 安装前安全停止 sidecar。
4. 更新完成后重新启动 PMBrain。
5. 新版本先执行数据库迁移，再启动服务并进行健康检查。

自动更新不会覆盖现有数据库地址、API Key、知识库目录或 MCP 配置。

## 配置、数据、备份和日志位置

- 新版配置：`%USERPROFILE%\.pmbrain\config.json`
- 旧版兼容配置：`%USERPROFILE%\.gbrain\config.json`
- 配置备份：当前配置目录下的 `backups\config`
- MCP 配置备份：当前配置目录下的 `backups\mcp`
- 桌面日志：`%APPDATA%\PMBrain Desktop\logs`
- PGLite 数据：首次向导中显示或选择的数据库路径

卸载桌面程序不会主动删除上述用户配置和知识库数据。重新安装后可以继续读取。

## 常见问题

### 点击图标后没有窗口

先在任务管理器确认是否存在 `PMBrain.exe`。1.0.22 起，再次点击图标会显示、恢复或重建主窗口；如果旧版本遗留进程仍占用单实例锁，可以结束旧进程后重新启动。

### 提示 `Cannot find module @electric-sql/pglite/vector`

这是 1.0.21 安装包遗漏 PGLite 运行时造成的打包问题，与用户数据库配置无关。请安装 1.0.22 或更高版本。

### 老用户为什么看到配置页

正常升级应直接进入管理台。只有首次使用、用户主动打开设置，或服务启动失败时才会显示桌面壳。启动失败时应优先查看恢复页和日志。

### Docker Postgres 无法连接

确认容器正在运行、端口映射正确、连接地址可用，并确认数据库已安装 `pgvector`。PMBrain 不会擅自切换为 PGLite，也不会自动重建数据库。

### 没有 API Key 能不能用

可以启动和管理本地数据，但依赖模型的向量化、对话和搜索增强功能不可用。后续补充 Key 即可，不需要重新安装。
